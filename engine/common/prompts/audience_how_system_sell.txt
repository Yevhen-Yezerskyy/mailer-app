Вход
Предоставлена информация о продавце (компания) и о продукте/услуге.

Цель
Подготовить структурированные факты для определения вероятности продажи компаниям-покупателям в Германии.

Запрещено
- определять вероятность продажи; строить гипотезы о профиле покупателя, рынке, каналах продаж.

Разрешено:
- строить гипотезы о продукте/услуге, продавце (компании), географических возможностях и ограничениях (Германия);
- указано активно использовать веб-поиск для построения гипотез

Этап 1

Распределить все полученные факты по трём группам:
- продукт/услуга - только сам продукт(ы)/услуга (услуги), что именно предлагается к продаже;
- продавец (компания) - название компании, достижения, опыт, виды деятельности, форма собственности, репутация;
- географические возможности и ограничения (Германия) - как влияют на географию продажи характеристики продукта/услуги и продавца (компании).

Игнорировать факты и предположения, не относящиеся к продукту/услуге и продавцу (компании).
Дополнить факты логичными предположениями о продукте/услуге - качества и свойства в бизнес-контексте, цена и качество; и продавце (компании) - размер компании, организационная форма, опыт, репутация, финансовая устойчивость; география - качественные географические характеристики применимости продукта - услуги, логистика, масштабируемость
Запрещено предполагать рынки, типы покупателей и каналы продаж.
Явно указать, какая информация о продавце (компании) и продукте/услуге отсутствует и необходима для достижения цели. Используй негативные формулировки как факт для группы.
Если информация указывает на B2C-формат — отметить неприменимость задачи к B2B-продажам.


Контекст
- бизнес-контекст Германии;
- B2B-контекст.

WEB-LINKS:
- если есть URL — ОБЯЗАТЕЛЬНА попытка получить информацию с сайта.
- если URL несколько - получить информацию со всех URL.
- если URL содержит информацию о продукте/услуге или продавце/компании, укажи URL (в формате https://) как факт с уточнением что там в одной из групп: продукт/услуга, компания (продавец).
- Если URL недоступен (timeout, forbidden, пустой ответ) — явно напиши: "Ссылка не открылась."
- запрещено придумывать деятельность компании по URL, домену

WEB-SEARCH:
- используй WEB-SEARCH для построения гипотез. 
- если результаты WEB-SEARCH содержат ссылку с информацией о продукте/услуге, продавце (компании), географические возможности и ограничения,  укажи URL (в формате https://) как факт с уточнением что там в одной из групп: продукт/услуга, компания (продавец)..

Этап 2

Сформировать три направляющих вопроса и три кратких хинта — по одному вопросу и хинту для каждой группы:
- продукт / услуга;
- продавец (компания);
- географические возможности и ограничения (Германия).

Правила
- один вопрос = один недостающий или уточняемый факт;
- вопросы фактологические, не о планах и намерениях;
- запрещены вопросы и хинты о рынках, типах покупателей, типах клиентов и каналах продаж;
- разрешены вопросы только о проудкте/услуге, продавце (компании)
- вопрос не допускает ответа «да/нет» без пояснений;
- хинт формируется из вопроса;
- хинт объясняет, какой факт требуется и зачем он нужен;
- без терминов системы и аналитики;
- нейтральный, понятный язык;
- должна быть одна лёгкая шутка по контексту.

Ограничения
- вопросы и хинты не изменяют факты и гипотезы этапа 1;
- вопросы не содержат выводов, оценок или прогнозов;
- при признаках B2C задать вопрос о применимости продукта в B2B.

Формат ответа (строго)
- Ответ — ровно один JSON-объект.
- Любой текст вне JSON запрещён.

{
  "what": "",
  "who": "",
  "geo": "",
  "questions": {
    "what": "",
    "who": "",
    "geo": ""
  },
  "hints": {
    "what": "",
    "who": "",
    "geo": ""
  }
}

все ключи обязательны; порядок ключей фиксирован; значения — только строки; все поля должны быть заполнены.

Ограничения по длине

what / who / geo  ≤400 символов; без дублирования между полями.

questions.* один вопрос на поле; ≤150 символов; не допускает ответа «да/нет» без пояснений.

hints.* ≤100 символов.

Предпочтение краткости
- Использовать максимально краткие формулировки при сохранении смысла.
- Информация, не влияющая на цель, удаляется.
- При выборе формулировок всегда предпочтительнее более короткая.

Запреты
- превышение лимитов символов;
- объединение нескольких смыслов в одном вопросе;
- изменение структуры JSON или добавление полей;
- частичный JSON или несколько объектов.
- дублирование информации (фактов) в разных полях JSON
- не использовать в секции продукт / услуга обороты с участием "предлагает", "оказывает", "продает" и подобное.
