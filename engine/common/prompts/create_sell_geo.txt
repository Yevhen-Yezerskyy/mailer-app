1) Общая задача
Сформируй три поля:
- `geo` — рабочее описание географических параметров для отбора бизнес-категорий и скоринга компаний-покупателей.
- `advice_answer` — короткий ответ на вопрос из ИНСТРУКЦИИ (если вопрос есть).
- `advice_question` — один уточняющий гипотеза-вопрос, который улучшает `geo`.

Вход:
- GEO
- ИНСТРУКЦИЯ

Общие правила:
- По умолчанию это B2B-задача для рынка Германии.
- Для решения задачи рекомендуется использовать веб-поиск.
- Если во входе есть ссылка, обязательно открой ее и используй данные страницы для дополнения `geo` и формирования `advice`.
- Не уходи в логистику, сертификацию, цены, контракты, экспорт/импорт, если этого нет во входе.
- Пиши только географические параметры, которые реально влияют на поиск категорий и компаний.
- Запрещены слова: "предполагается", "уточняется", "возможно", "вероятно", "может быть", "ориентировочно", "и др.", "и т.п.".
- Пиши прямо, без воды и служебных комментариев.

2) Формирование `geo` (и обработка GEO)
Правила обновления:
- Если GEO короткий, на первом ходе разверни рабочую гипотезу.
- На следующих ходах меняй только то, что изменил пользователь, или то, что прямо требует ИНСТРУКЦИЯ.
- На втором и следующих ходах новые гипотезы не встраивай в `geo`; формулируй их в `advice_question`.
- Удаленное пользователем не возвращай.
- Если изменений нет, верни `geo` без изменений.

Структура `geo` (обязательно):
1. Основная география
- Страны, регионы, города, зоны или тип территории, где нужен поиск.

2. Приоритеты поиска
- Какие географические направления важнее.
- Какие географические признаки влияют на отбор категорий и компаний.

3. Ограничения
- Что исключать или ограничивать в географии поиска, если это есть во входе.

Требования к `geo`:
- Без дублей по смыслу.
- Пиши сухо и предметно.
- Оставляй только признаки, которые влияют на отбор категорий и профиль покупателя.
- Длина: 500-1200 символов.

3) Формирование `advice` (и обработка ИНСТРУКЦИЯ)
Правила:
- Если ИНСТРУКЦИЯ требует изменить `geo`, примени это в `geo`.
- Если ИНСТРУКЦИЯ содержит вопрос, ответ дай в `advice_answer`.
- Если вход неоднозначный, вопрос в `advice_question` должен фиксировать географию поиска (это исключение).

Формат `advice_answer` и `advice_question`:
- `advice_answer`: если в ИНСТРУКЦИИ есть вопрос, дай короткий ответ (не более 2 предложений). Иначе пустая строка.
- `advice_question`: возьми текущий `geo` как базу, используй web search и сформируй ровно 1 новый гипотеза-вопрос.
- Вопрос должен добавлять новый географический признак для отбора категорий/покупателей, которого нет в `geo`.
- Не повторяй уже написанное в `geo`.
- В вопросе дай 2-4 варианта ответа в скобках.
- В скобках укажи 2-4 новые варианты ответа, которых нет в `geo`.
- Без вводных абзацев и рекомендаций.
- Длина: 90-350 символов.

Самопроверка перед ответом:
- Удали запрещенные слова из `geo`, `advice_answer`, `advice_question`.
- Проверь, что `geo` содержит 3 части: Основная география / Приоритеты поиска / Ограничения.
- Проверь, что варианты ответа в скобках не повторяют слова и смыслы из `geo`.
- Проверь, что `advice_question` содержит ровно 1 вопрос и он помогает уточнить географию поиска.

Ответ строго JSON:
{
  "geo": "...",
  "advice_answer": "...",
  "advice_question": "..."
}
