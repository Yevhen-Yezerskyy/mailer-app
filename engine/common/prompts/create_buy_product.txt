1) Общая задача
Сформируй три поля:
- `product` — рабочее описание того, что нужно купить, заказать или отдать на подряд в форме, удобной для отбора бизнес-категорий, где есть поставщики / подрядчики, и для дальнейшего ранжирования уже отобранных поставщиков / подрядчиков.
- `advice_answer` — короткий ответ на вопрос из КОМАНДЫ (если вопрос есть).
- `advice_question` — один уточняющий вопрос-гипотеза, который улучшает `product`.

Вход:
- ПРОДУКТ
- КОМАНДА

Разбор КОМАНДЫ:
- КОМАНДА может содержать `УКАЗАНИЕ`, `ВОПРОС` или оба сразу.
- `УКАЗАНИЕ` меняет `product`.
- `ВОПРОС` не меняет `product`; ответ дай в `advice_answer`.

Общие правила:
- По умолчанию это B2B-задача для рынка Германии.
- В этом prompt `купить` и `заказать` означают одно и то же.
- Главная цель `product`:
  1. отобрать категории, где есть поставщики / подрядчики для этого продукта, услуги или работ;
  2. ранжировать уже отобранных поставщиков / подрядчиков.
- Не смещай описание в сторону продажи, покупателей или клиентского спроса.
- Не выдумывай SaaS/платформу/бренд/стартап, если этого нет во входе.
- Для решения задачи рекомендуется использовать веб-поиск.
- Если во входе есть ссылка, обязательно открой ее и используй данные страницы для дополнения `product` и `advice_question`.
- Не инициируй темы: логистика, отгрузка, склады, порты/узлы, сертификация, юр. условия, цены, контракты, экспорт/импорт.
- Не используй географические соображения: страны, регионы, города, радиусы, расстояния, близость/удаленность, территориальный охват, локализация.
- Запрещены слова: "предполагается", "уточняется", "возможно", "вероятно", "может быть", "ориентировочно", "и др.", "и т.п.".
- Пиши прямо, без воды и служебных комментариев.

2) Формирование `product` (и обработка ПРОДУКТ)
Правила обновления:
- Если ПРОДУКТ короткий, на первом ходе разверни рабочую гипотезу.
- На следующих ходах меняй только то, что изменил пользователь, или то, что прямо требует `УКАЗАНИЕ` из КОМАНДЫ.
- Любые новые сведения, которые пользователь прямо добавил в `ПРОДУКТ` или прямо указал добавить в `КОМАНДЕ`, считай фактами для обновления `product`, а не гипотезами.
- На втором и следующих ходах новые гипотезы, которые модель выводит сама без прямого указания пользователя, не встраивай в `product`; формулируй их в `advice_question`.
- Удаленное пользователем не возвращай.
- Если изменений нет, верни `product` без изменений.
- `ВОПРОС` из КОМАНДЫ сам по себе не должен менять `product`.

Поле `product` — это текст из 3 секций.
1. Что нужно купить / заказать?
- Кратко и сухо зафиксируй объект закупки / заказа.
- Дай только те характеристики, которые помогают понять, в каких отраслях и бизнес-категориях искать поставщиков / подрядчиков и как потом их ранжировать.

2. К каким отраслям, типам бизнеса и бизнес-категориям относятся поставщики / подрядчики, у которых это покупают / заказывают?
- Укажи реальные отрасли, типы бизнеса и бизнес-категории поставщиков / подрядчиков, у которых обычно покупают / заказывают этот продукт, услугу или работы.
- Это раздел про поставщиков / подрядчиков, а не про применение продукта и не про покупателей со стороны клиента.
- Не пиши, где продукт используется, для каких задач он нужен и кто это покупает как клиент.
- Пиши в терминах отраслей, типов бизнеса и бизнес-категорий поставщиков / подрядчиков.
- Для каждой отрасли, типа бизнеса и категории укажи важность или приоритет.

3. Какие поставщики / подрядчики подходят лучше всего?
- Укажи критерии поставщиков / подрядчиков, которые подходят лучше всего, нейтрально или хуже всего.
- Это могут быть специализация, размер, производственный профиль, формат работ, уровень сложности, опыт выполнения похожих задач или другие признаки, которые реально помогают ранжировать поставщиков / подрядчиков.
- Для каждого критерия укажи важность или приоритет.

Требования к `product`:
- Описывай объект закупки / заказа так, чтобы по нему можно было искать и ранжировать категории, где есть поставщики / подрядчики для этого продукта, услуги или работ.
- Без географии в любом виде.
- Без дублей по смыслу.
- Пиши сухо и предметно: без вводных определений, энциклопедических описаний и общей теории.
- Оставляй только признаки, которые влияют на отбор категорий и ранжирование поставщиков / подрядчиков.
- Не добавляй технические, рецептурные, качественные и эксплуатационные детали, если они не меняют отбор категорий и ранжирование поставщиков / подрядчиков.
- Длина: до 1100 символов.

3) Формирование `advice_answer`
- Если в КОМАНДЕ есть `ВОПРОС`, дай короткий ответ (не более 2 предложений) с учетом контекста. Иначе верни пустую строку.

4) Формирование `advice_question`
- Сформируй ровно 1 новый вопрос-гипотезу по текущему `product` с учетом контекста и веб-поиска.
- Вопрос должен добавлять новый признак, полезный для дальнейшего отбора категорий и ранжирования поставщиков / подрядчиков.
- Если разные ответы не меняют категории поиска и не влияют на ранжирование поставщиков / подрядчиков, такой вопрос не задавай.
- Вопрос должен быть ориентирован на задачу поиска поставщика / подрядчика.
- Дай 2-4 новых варианта ответа в скобках.
- Без вводных абзацев и рекомендаций.
- Длина: до 350 символов.

Самопроверка перед ответом:
- Удали запрещенные слова из `product`, `advice_answer`, `advice_question`.
- Проверь, что значение поля `product` — это текст из 3 секций: что нужно купить / заказать / отдать на подряд / к каким отраслям, типам бизнеса и бизнес-категориям обычно относятся поставщики / подрядчики, у которых это покупают / заказывают / какие поставщики / подрядчики подходят лучше всего.
- Проверь, что `product` не смещен в сторону продаж, покупателей или клиентского спроса.
- Проверь, что `product` не содержит признаков, которые не влияют на отбор категорий и ранжирование поставщиков / подрядчиков.
- Проверь, что `advice_answer` отвечает только на вопрос из КОМАНДЫ; если вопроса нет, верни пустую строку.
- Проверь, что `advice_question` содержит ровно 1 вопрос о признаке, полезном для отбора категорий и ранжирования поставщиков / подрядчиков.
- Проверь, что варианты ответа в `advice_question` не повторяют `product` и тоже полезны для отбора категорий и ранжирования поставщиков / подрядчиков.

Ответ строго JSON:
{
  "product": "...",
  "advice_answer": "...",
  "advice_question": "..."
}
