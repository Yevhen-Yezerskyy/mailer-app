Вход: Информация: продукт / услуга, которая покупается; покупатель (компания), который покупает; географические возможности и ограничения (Германия).

Выход:
Формат ответа (строго)
- Ответ — ровно один JSON-объект.
- Любой текст вне JSON запрещён.

{
  "Seller": "",
  "Branches": "",
  "Geo": "",
}

Пояснения: Сформулированы три задачи - Seller, Branches, Geo. Результатом выполнения каждой задачи надо заполнить соответствующие поле JSON. 
Если вход содержит URL, посети их. Для определения критериев оценки используй веб-поиск.

Задачи:
Задача 1. Seller.

Сформулируй 5 критериев, по которым система будет определять вероятность успешной сделки (покупка, приобретение) с конкретной компанией. 
Эти критерии будут использоваться на следующем шаге для ранжирования найденных компаний по вероятности конверсии.

Требования к ответу:
- текст. Вставить в общий JSON в поле Seller.
– выдай ровно 5 критериев;
– каждый критерий — не длиннее 120 символов (без учёта важности);
– каждому критерию присвой важность от 1 до 5, где 1 — самый важный;
– критерии должны быть отсортированы по важности (от 1 к 5); Важность = номер в списке.
– критерии должны описывать принципы оценки вероятности сделки, а не сами компании;
– ответ должен быть структурирован и пригоден для прямого использования как инструкция в следующем этапе.

Формат ответа (строго соблюдать):

1 – Критерий
2 – Критерий
3 – Критерий
4 – Критерий
5 – Критерий


Запрещено:
– упоминать конкретные компании;
– приводить примеры;
– выполнять саму оценку;
– формулировать больше или меньше 5 критериев.
