Входные данные:
- описание продукта и продавца (ниже)
- подсказка для оценки бранчей (ниже)
- список КАНДИДАТОВ branchen (каждый элемент содержит id и name).

Задача:
Для каждого бранча из списка оценить вероятность того, что данный бранч:
- станет покупателем продукта,
- заплатит деньги продавцу в B2B-сделке.

Рейтинг:
- rate — целое число от 1 до 100.
- меньший rate означает более высокую вероятность покупки и оплаты продавцу.
- больший rate означает более низкую вероятность покупки и оплаты продавцу.
- намеренно используй весь диапазон рейтинга 1–100.

При оценке учитывай все факторы, влияющие на вероятность покупки, включая:
- типичность покупки подобных продуктов данным бранчем,
- способность и готовность платить,
- скорость принятия решения и заключения сделки.

Правила:
- Используй ТОЛЬКО id и name из переданного списка кандидатов.
- Никаких новых branchen не добавляй.
- Каждый бранч оценивай относительно всей совокупности бранчей
немецких «Жёлтых страниц» (≈800 бранчей), а не только относительно переданного списка.
- Не выравнивай оценки и не избегай низких значений.

Формат ответа (СТРОГО, только JSON, без текста, комментариев и markdown):
[
{"id": 123, "name": "Fensterbau", "rate": 7},
{"id": 456, "name": "Metallbau", "rate": 12}
]

Каждый элемент ответа должен быть объектом строго с ключами:
id, name, rate.

Используй веб-поиск, если данных недостаточно.
