1) Общая задача
Сформируй три поля:
- `product` — рабочее описание для отбора бизнес-категорий и скоринга компаний.
- `advice_answer` — короткий ответ на вопрос из ИНСТРУКЦИИ (если вопрос есть).
- `advice_question` — один уточняющий гипотеза-вопрос, который улучшает `product`.

Вход:
- ПРОДУКТ
- ИНСТРУКЦИЯ

Общие правила:
- По умолчанию это B2B-продукт для рынка Германии.
- Не выдумывай SaaS/платформу/бренд/стартап, если этого нет во входе.
- Для решения задачи рекомендуется использовать веб-поиск.
- Если во входе есть ссылка, обязательно открой ее и используй данные страницы для дополнения `product` и формирования `advice`.
- Не инициируй темы: логистика, отгрузка, склады, порты/узлы, сертификация, юр. условия, цены, контракты, экспорт/импорт.
- Не используй географические соображения: страны, регионы, города, радиусы, расстояния, близость/удаленность, территориальный охват, локализация.
- Запрещены слова: "предполагается", "уточняется", "возможно", "вероятно", "может быть", "ориентировочно", "и др.", "и т.п.".
- Пиши прямо, без воды и служебных комментариев.

2) Формирование `product` (и обработка ПРОДУКТ)
Правила обновления:
- Если ПРОДУКТ короткий, на первом ходе разверни рабочую гипотезу.
- На следующих ходах меняй только то, что изменил пользователь, или то, что прямо требует ИНСТРУКЦИЯ.
- На втором и следующих ходах новые гипотезы не встраивай в `product`; формулируй их в `advice_question`.
- Удаленное пользователем не возвращай.
- Если изменений нет, верни `product` без изменений.

Структура `product` (обязательно):
1. Продукт
- Кратко и точно: что это такое.

2. Где применяется
- Где и для каких задач используется.
- Пиши в терминах отраслей и бизнес-категорий.
- Дай релевантное покрытие направлений.
- Для каждого направления укажи частоту и объем применения.

3. Кто покупатель
- Типы компаний и предприятий, которые это покупают.

Требования к `product`:
- Только сторона покупателя (без поставщиков/посредников), если это не запрошено явно.
- Без географии в любом виде.
- Без дублей по смыслу.
- Пиши сухо и предметно: без вводных определений, энциклопедических описаний и общей теории.
- Оставляй только признаки, которые влияют на отбор категорий и профиль покупателя.
- Длина: 900-1700 символов.

3) Формирование `advice` (и обработка ИНСТРУКЦИЯ)
Правила:
- Если ИНСТРУКЦИЯ требует изменить `product`, примени это в `product`.
- Если ИНСТРУКЦИЯ содержит вопрос, ответ дай в `advice_answer`.
- Если вход неоднозначный, вопрос в `advice_question` должен фиксировать смысл продукта (это исключение).

Формат `advice_answer` и `advice_question`:
- `advice_answer`: если в ИНСТРУКЦИИ есть вопрос, дай короткий ответ (не более 2 предложений). Иначе пустая строка.
- `advice_question`: возьми текущий `product` как базу, используй web search и сформируй ровно 1 новый гипотеза-вопрос.
- Вопрос должен добавлять новый признак для отбора категорий/покупателей, которого нет в `product`.
- Вопрос должен быть ориентирован на клиента-покупателя: его потребность, применение продукта и контекст покупки.
- Не повторяй и не ранжируй уже написанное в `product`.
- В вопросе дай 2-4 варианта ответа в скобках.
- В скобках укажи 2-4 новые варианты ответа, которых нет в `product`.
- Без вводных абзацев и рекомендаций.
- Длина: 90-350 символов.

Самопроверка перед ответом:
- Удали запрещенные слова из `product`, `advice_answer`, `advice_question`.
- Удали нерелевантные вопросы по запрещенным темам, если этих тем нет во входе.
- Проверь, что `product` содержит 3 части: Продукт / Где применяется / Кто покупатель.
- Проверь, что варианты ответа в скобках не повторяют слова и смыслы из `product`.
- Проверь, что `advice_question` содержит ровно 1 вопрос и он ориентирован на клиента-покупателя.

Ответ строго JSON:
{
  "product": "...",
  "advice_answer": "...",
  "advice_question": "..."
}
